# Roblox Route Bot

Внешняя (без инъекции в игру) программа на C#/WPF: рисуешь маршрут — персонаж бежит по нему,
с обратной связью против дрейфа. Управление через `SendInput` (WASD), позиция считывается с экрана.

> Анти-чит Hyperion/Byfron ловит инъекции и чтение памяти процесса. Здесь ни того, ни другого:
> только ввод уровня ОС + чтение картинки с экрана. Тем не менее у конкретной игры может быть
> **серверный** детект ботоподобного движения — это уже на твой риск.

## Запуск

```powershell
dotnet run -c Release
```
или собрать и запустить exe:
```powershell
dotnet build -c Release
.\bin\Release\net8.0-windows\RobloxRouteBot.exe
```

## Как пользоваться

1. **Окно игры** — впиши часть заголовка окна (по умолчанию `Roblox`).
2. **Нарисуй маршрут** — зажми ЛКМ на чёрном поле и веди мышью. Линия = путь.
3. **Глаза** — выбери режим (см. ниже).
4. **Старт** — пойдёт обратный отсчёт 2 сек: за это время **кликни в окно игры**
   (ввод идёт в активное окно!). Камеру в игре держи неподвижно.
5. **Стоп** — остановить, все клавиши отпускаются.

## Режимы «глаз» (как бот понимает, где персонаж)

| Режим | Что делает | Дрейф |
|---|---|---|
| **Open-loop** | слепое воспроизведение по таймингу; дот идёт по маршруту в симуляции | копится (база для сравнения) |
| **Оптический поток** | меряет реальный сдвиг картинки между кадрами → closed-loop | почти нет, но медленно ползёт без «якоря» |

- **Open-loop:** параметр `Скорость` (единиц канваса/сек) должен примерно совпадать с реальной
  скоростью персонажа, иначе тайминг разъедется. Это базовый режим — он **будет** дрейфовать.
- **Оптический поток:** параметр `px→world` (масштаб: сколько единиц канваса на 1 пиксель сдвига)
  нужно **откалибровать** под игру. Пока он на глаз — это следующий шаг настройки вживую.

## Архитектура

```
[ScreenCapture] → [IPositionProvider: глаза] → [PurePursuitController] → [MovementMapper] → [InputSender]
                         ↑__________ BotEngine: цикл ~60 Гц на отдельном потоке __________↓
```

- `Vision/` — захват экрана, провайдеры позиции (open-loop, оптический поток; минимапа/OCR — TODO).
- `Core/` — контроллер следования, маппер направления→WASD, движок-цикл.
- `Input/` — `SendInput` скан-кодами, высокоточный таймер.
- `Native/` — поиск окна, его координаты.

## Что в v1 работает, а что ещё нет

**Работает:** рисование маршрута, точный тайминг ввода, фикс-камера + WASD-стрейф (нет поворотной
ошибки), open-loop полностью, оптический поток как closed-loop.

**Нужна доводка вживую (следующие шаги):**
- Калибровка оптического потока под игру (`px→world`, инверсия/поворот осей).
- «Ре-зеро»-якоря (упор в стену) — чтобы убрать остаточный дрейф потока на длинных кругах.
- Провайдер минимапы/координат (OCR) — абсолютная позиция, истинный ноль дрейфа, если игра их
  показывает.
- Режим записи живого прохождения (сейчас только рисование).
- Произвольные углы через ШИМ по направлению (сейчас 8 сторон).
